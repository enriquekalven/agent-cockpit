# Optimized Agent Stack - LLM Context

This file provides a machine-readable summary of the **Optimized Agent Stack** for LLMs and AI Agents.

## Purpose
A rapid-development framework for creating agentic interfaces with built-in **Agent Ops** (Day 2 operations) for Google Cloud.

## Core Modules
- Renderer: `src/a2ui/A2UIRenderer.tsx` - Handles JSON-to-UI conversion.
- Optimizer: `src/backend/optimizer.py` - Interactive CLI for cost & perf audit.
- Shadow Mode: `src/backend/shadow/router.py` - Side-by-side agent comparison.
- Semantic Cache: `src/backend/cache/semantic_cache.py` - "Hive Mind" cost reduction.
- Red Team Eval: `src/backend/eval/red_team.py` - Adversarial security audit.
- MCP Hub: `src/backend/ops/mcp_hub.py` - Standardized tool connectivity hub.
- Ops Dashboard: `src/components/OpsDashboard.tsx` - Visual control room / Flight Recorder.

## Key Viral Features
1. **Shadow Mode**: Deploy experimental models (v2) in parallel with production (v1) with zero user impact.
2. **Hive Mind Cache**: Drop-in semantic caching to cut LLM costs and latency.
3. **Red Team CI/CD**: Automatic security vetting to prevent prompt injections.
4. **Flight Recorder**: Visual scrubbable debugging of agent thought chains.
5. **MCP Tool Hub**: Standardized tool connectivity and usage optimization.
6. **Zero-Code RAG**: Auto-indexing of the `src/knowledge/` folder.

## Quick Start
1. `npm install`
2. `make audit` - Run optimization audit.
3. `make red-team` - Run security audit.
4. `make dev` - Start local environment.
5. `make deploy-prod` - Full GCP deployment (Cloud Run).

## Connectivity
The playground can connect to a local ADK agent running on `localhost:8000`.
To enable, toggle "Connect to Local ADK Agent" in the Playground.
The Ops Dashboard is available at `/ops`.
